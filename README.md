# PRODIGY_DS_04
📊 Social Media Sentiment Analysis

![Python](https://img.shields.io/badge/python-3.9%2B-blue)
![License](https://img.shields.io/badge/license-MIT-green)
![Status](https://img.shields.io/badge/status-active-brightgreen)

Data cleaning and exploratory data analysis (EDA) on social media posts to
understand **public opinion and attitudes toward specific brands, games, and
topics** — built for **Prodigy InfoTech's Data Science Internship, Task 4**.

Given a raw, messy stream of social media posts labeled by entity (brand/topic)
and sentiment, this project cleans the text, explores the data, and produces a
set of visualizations that answer questions like:

- What's the overall public mood — mostly positive, negative, or mixed?
- Which brands/topics get talked about the most?
- Which ones are viewed favorably vs. unfavorably?
- What words actually drive positive vs. negative sentiment?

✨ Features

- **Robust cleaning pipeline** — handles missing text, duplicate posts, inconsistent
  label casing, URLs, @mentions, hashtags, HTML entities, and elongated words
  ("soooo good" → "soo good"), with **zero required external NLP downloads**
  (no `nltk.download()` calls that can fail offline)
- **Entity-level sentiment analysis** — not just "is this positive or negative"
  but "how does the public feel about *each specific brand/topic*"
- **A "Net Sentiment Score"** (`%Positive − %Negative`) to rank entities from
  most to least favorably viewed
- **Optional bonus check** — cross-validates the dataset's human labels against
  VADER, a lexicon-based sentiment scorer, when installed
- **Runs two ways**: an exploratory Jupyter notebook, or a single CLI command
  (`run_pipeline.py`) that saves all figures + cleaned data — useful for CI or
  quick re-runs
- **Ships with a small synthetic sample dataset** so everything works
  immediately after cloning — no download required to try it out
- **Unit tested** cleaning logic (`tests/`)

📦 Dataset

**Source:** [Prodigy-InfoTech/data-science-datasets — Task 4](https://github.com/Prodigy-InfoTech/data-science-datasets/tree/main/Task%204)
(originally the [Twitter Entity Sentiment Analysis](https://www.kaggle.com/datasets/jp797498e/twitter-entity-sentiment-analysis) dataset on Kaggle)

Each row is one social media post about a specific entity, labeled `Positive`,
`Negative`, `Neutral`, or `Irrelevant`. Full details, schema, and download
instructions are in **[`data/README.md`](data/README.md)**.

> The full raw CSV is ~10 MB and isn't committed to this repo. A small
> synthetic sample with the identical schema is bundled instead so you can run
> everything immediately — see `data/README.md` for the one-line command to
> pull the real dataset.

⚙️ Installation

```bash
git clone https://github.com/<your-username>/social-media-sentiment-analysis.git
cd social-media-sentiment-analysis
pip install -r requirements.txt
```

Requires Python 3.9+. See `requirements.txt` — everything except `jupyter`,
`wordcloud`, and `vaderSentiment` is required; those three are optional and
the code degrades gracefully without them.

**Want to analyze the full ~74,000-row dataset instead of the bundled sample?**

```bash
python data/download_data.py
```

🚀 Usage

**Option A — Notebook (recommended for exploring):**

```bash
jupyter notebook notebooks/01_sentiment_eda.ipynb
```

Every cell is already pre-run in this repo (on the bundled sample) so you can
also just open and read it on GitHub without running anything.

**Option B — Command line (recommended for a quick full re-run):**

```bash
python run_pipeline.py
```

This loads the data (full dataset if present in `data/raw/`, sample
otherwise), cleans it, prints summary stats, and saves every chart to
`outputs/figures/` plus the cleaned dataset to `outputs/cleaned_data.csv`.

🧹 The cleaning pipeline

Implemented once in `src/data_cleaning.py` and shared by both the notebook and
`run_pipeline.py`, so they can never drift out of sync:

1. **Standardize labels** — trims whitespace, fixes casing (`"positive"` → `"Positive"`)
2. **Drop invalid rows** — missing text, exact duplicates, or a sentiment label
   outside the 4 valid classes
3. **Clean the text** — lowercase → strip URLs/@mentions/HTML entities/hashtag
   symbols/punctuation/digits → collapse repeated letters → remove stopwords
4. **Add derived features** — `clean_text`, `char_count`, `word_count`
